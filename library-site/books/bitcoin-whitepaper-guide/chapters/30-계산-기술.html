11. Calculation — Technical Understanding

Proving the Whitepaper's Formulas

The Attacker's Catch-Up Probability — Binomial Random Walk and Recurrence

Let p be the probability that honest miners find the next block first, and q = 1 − p be the attacker's probability. Q(z) denotes the probability that an attacker currently z blocks behind eventually catches up. This is not simply q^z, because honest miners keep mining while the attacker works. The number of victories the attacker needs changes dynamically — this is the essence of the binomial random walk.

The recurrence relation is:

Q(z) = p · Q(z+1) + q · Q(z−1)

Defining b(n) = Q(n+1) − Q(n), b(n) forms a geometric sequence with ratio p/q. Applying the geometric series formula expresses Q(z) in terms of two unknowns. The boundary conditions Q(0) = 1 (already caught up) and Q(N) = 0 (gives up after falling N blocks behind) determine the unknowns. Taking the limit N → ∞ yields:

  • If q < p: Q(z) = (q/p)^z
  • If q ≥ p: Q(z) = 1

This is the formula in the whitepaper. When the attacker controls more than half the hash power, they will eventually catch up with probability 1 regardless of how many blocks separate them — the origin of the term "51% attack." When q < p, the probability decays exponentially with z. For q = 0.2 (attacker at 20%): Q(1) = 25%, Q(2) = 6.25%, Q(3) = 1.56%, Q(6) ≈ 0.024%.

In 2022, Bitcoin's total energy consumption for mining was approximately 55 billion kWh — more than all computers in the United States combined. A 51% attacker would need to match this output single-handedly, which is beyond the capability of any organization.

Applying the Poisson Distribution

Now consider how long a recipient must wait. The recipient reveals the payment address only at the moment of the transaction, preventing the attacker from pre-building an advantageous chain. Once the transaction is broadcast, the attacker secretly mines a parallel chain.

While honest miners add z blocks, the number of blocks the attacker manages to mine, X, follows a **Poisson distribution** with mean λ = z · (q/p). With z = 8, q = 0.2, p = 0.8: λ = 8 × 0.25 = 2. On average the attacker mines 2 blocks while honest miners mine 8.

The Poisson distribution arises as the limit of the binomial distribution B(n, p) as n → ∞ and p → 0 with np = λ constant. Because time is continuous and can be subdivided indefinitely, the probability mass function converges to P(X = k) = e^(−λ) · λ^k / k!, derived using the definition of Euler's number e = lim(1 + λ/n)^n.

Computing the Probability of Mallory's Fraud Succeeding

For the attack to succeed, the attacker must catch up from whatever deficit remains after the confirmation period. If the attacker mined k blocks while honest miners mined z blocks (k < z), the attacker still needs to overcome a deficit of z − k. If k ≥ z the attack already succeeded. The total success probability is:

P = Σ(k=0 to z−1) [e^(−λ) · λ^k / k! · (q/p)^(z−k)] + Σ(k=z to ∞) [e^(−λ) · λ^k / k!]

The whitepaper rewrites this to avoid summing an infinite series, producing the computationally tractable form:

P = 1 − Σ(k=0 to z) [e^(−λ) · λ^k / k! · (1 − (q/p)^(z−k))]

Code Implementation and Its Relation to the Formula

The C function `AttackerSuccessProbability(double q, int z)` attached to the whitepaper implements this formula directly. It initializes `sum = 1.0` (incorporating the tail k > z), computes the Poisson mass for each k from 0 to z, and subtracts the weighted catch-up probability. The phrase "rearranging to avoid summing the infinite tail of the distribution" in the whitepaper refers exactly to this reformulation that makes the computation finite and exact.

Interpreting Satoshi's Numerical Results

The whitepaper tabulates P for q = 0.1 and q = 0.3 across various z. For q = 0.1: z = 5 gives P ≈ 0.000914; z = 6 gives P ≈ 0.000243. For q = 0.3: z = 24 gives P ≈ 0.000248. The minimum z satisfying P < 0.001 is: q = 0.10 → z = 5; q = 0.15 → z = 8; q = 0.20 → z = 11; q = 0.25 → z = 15; q = 0.30 → z = 24; q = 0.35 → z = 41; q = 0.40 → z = 89; q = 0.45 → z = 340.

In Mallory's concrete example — controlling 20% of hash power, with 8 honest blocks mined during the 80-minute transaction — the fraud success probability is approximately 0.39%. Mallory fails with 99.61% probability. Since controlling 20% of global Bitcoin hash power is itself nearly impossible, real-world attack success probabilities approach zero. This mathematical certainty, embedded in the whitepaper's formulas and confirmed by the attached code, is the quantitative foundation of Bitcoin's security model.

11. 계산 — 기술적 이해

백서의 수식 증명

공격자가 따라잡을 확률 — 이항 랜덤 워크와 점화식

정직한 체인과 공격자 체인 간의 경쟁은 이항 랜덤 워크다. 정직한 채굴자들이 다음 블록을 먼저 연결할 확률을 p, 공격자가 먼저 연결할 확률을 q(= 1 − p)라 하자. 공격자가 z블록 뒤처진 상황에서 정직한 체인을 따라잡을 확률 Q(z)는 단순히 q를 z번 연속 이기는 확률이 아니다. 그 사이에도 정직한 채굴자들이 계속 블록을 연결하기 때문이다. 따라잡기 위해 필요한 승리 횟수는 매 순간 변하므로 이 문제는 진정한 이항 랜덤 워크, 즉 도박꾼의 파산 문제가 된다.

Q(z)에 관한 점화식은 다음과 같다. 다음 블록을 정직한 노드가 먼저 연결하면 공격자는 z+1블록을 따라잡아야 하고, 공격자가 먼저 연결하면 z-1블록만 따라잡으면 된다.

Q(z) = p · Q(z+1) + q · Q(z−1)

이를 수열로 표현하면 b(n) = Q(n+1) − Q(n)으로 정의할 때 b(n)은 등비수열이 되고, 공비는 p/q가 된다. 등비수열의 합 공식을 적용하면 Q(z)를 두 미지수로 표현할 수 있다. 경계 조건 Q(0) = 1(이미 따라잡은 상태)과, 공격자가 N블록 이상 뒤처지면 포기한다는 가정 Q(N) = 0을 도입하여 풀고, 마지막에 N → ∞ 극한을 취하면 다음 결과를 얻는다.

  • q < p (공격자가 소수)인 경우: Q(z) = (q/p)^z
  • q ≥ p (공격자가 과반 이상)인 경우: Q(z) = 1

이 결과가 백서에 나오는 수식이다. 공격자가 전체 해시 파워의 51% 이상을 확보하면 아무리 많은 블록이 쌓여도 언젠가 반드시 따라잡을 확률이 100%가 된다. 반대로 소수를 차지할 때는 뒤처진 블록 수에 지수적으로 비례하여 확률이 감소한다. 공격자가 20%의 해시 파워를 가진다면 1블록 뒤처진 경우 (0.2/0.8)^1 = 25%, 2블록은 6.25%, 3블록은 1.56%, 6블록은 약 0.024%로 급감한다.

2022년 기준 비트코인 채굴 총 에너지 소비량은 약 550억 kWh로, 미국 내 모든 컴퓨터가 사용하는 에너지보다 많다. 51% 공격을 100% 성공시키려면 이 에너지의 전부를 혼자 운용해야 한다. 이는 어떤 거대 기관도 불가능한 수준이다.

맬러리의 거래 사기 프로젝트 — 푸아송 분포 적용

이제 수취인이 얼마나 오래 기다려야 하는지를 계산하자. 수취인은 서명 직전에 공개키를 알려줌으로써 공격자가 미리 유리한 상황을 준비하는 것을 막는다. 거래가 전송되면 공격자는 병렬 체인에서 몰래 작업을 시작한다.

정직한 채굴자들이 z개의 블록을 연결하는 동안 공격자가 연결한 블록 수 X는 푸아송 분포를 따른다. 기댓값은 λ = z · (q/p)이다. 예를 들어 공격자 해시 파워가 20%(q = 0.2), 정직한 채굴자가 80%(p = 0.8)이고, 정직한 채굴자가 8개의 블록을 연결했다면, λ = 8 × (0.2/0.8) = 2다. 즉 공격자는 평균 2개의 블록을 연결한다.

푸아송 분포는 이항분포 B(n, p)에서 n → ∞, p → 0, np = λ(일정)인 극한으로 유도된다. 시간은 연속적이므로 채굴 완료 사건의 단위 시간을 무한히 쪼갤 수 있고, 이 극한에서 확률질량함수는 P(X = k) = e^(−λ) · λ^k / k!이 된다. 자연상수 e의 정의 lim(1 + λ/n)^n = e^λ와 극한 계산을 거치면 이 공식이 정확히 유도된다.

맬러리의 사기 성공 확률

공격자가 성공하려면 정직한 채굴자들이 z개의 블록을 연결하는 동안 k개(k ≤ z)의 블록을 연결한 뒤, 남은 z − k블록의 격차를 따라잡아야 한다. k ≥ z이면 이미 성공이다. 따라서 전체 성공 확률 P는 다음과 같다.

P = Σ(k=0 to z−1) [e^(−λ) · λ^k / k! · (q/p)^(z−k)] + Σ(k=z to ∞) [e^(−λ) · λ^k / k!]

백서에서는 무한급수의 꼬리를 컴퓨터가 계산할 수 있도록 정리하여 다음과 같이 변환한다.

P = 1 − Σ(k=0 to z) [e^(−λ) · λ^k / k! · (1 − (q/p)^(z−k))]

이를 C 언어 코드로 구현한 것이 백서에 첨부된 AttackerSuccessProbability 함수다. 이 함수는 q와 z를 입력받아 위 합산을 수행한다.

맬러리의 구체적인 예시로 돌아가면, 전체 해시레이트의 20%를 운용하는 맬러리가 80분(정직한 채굴자들이 8개 블록 연결) 동안 사기를 시도할 때 성공 확률은 약 0.39%다. 즉 99.61%의 확률로 맬러리의 사기는 실패한다. 세계 비트코인 해시레이트의 20%나 운용할 수 있는 능력 자체가 현실에서 거의 불가능하므로, 실제 공격 성공 확률은 사실상 0에 수렴한다.

사토시 나카모토의 계산 결과 해석

백서에는 q=0.1과 q=0.3 두 가지 경우에 대해 z값에 따른 성공 확률 P가 수록되어 있다. q=0.1인 경우 z=5에서 P ≈ 0.000914, z=6에서 P ≈ 0.000243이다. q=0.3인 경우 z=24에서 P ≈ 0.000248이다. 이를 통해 공격자 비율이 낮을수록, 그리고 기다리는 블록 수가 많을수록 공격이 기하급수적으로 어려워짐을 확인할 수 있다.

P < 0.1%를 만족하는 최소 블록 수는 q=0.10일 때 z=5, q=0.15일 때 z=8, q=0.20일 때 z=11, q=0.25일 때 z=15, q=0.30일 때 z=24, q=0.35일 때 z=41, q=0.40일 때 z=89, q=0.45일 때 z=340이다. 이 계산들이 '6 컨펌 규칙'의 수학적 근거를 형성한다. 현실적으로 어떤 단일 공격자도 전체 해시 파워의 10%를 넘기기 어려우므로, 일반적인 거래에서 6 컨펌이면 충분히 안전하다.

코드 구현과 수식의 관계

백서에 첨부된 C 언어 코드 AttackerSuccessProbability(double q, int z)는 위에서 유도한 공식을 그대로 구현한 것이다. 코드는 먼저 λ = z × (q/p)를 계산한 뒤, k = 0부터 z까지 각 항에 대해 푸아송 확률질량함수 값 e^(−λ) × λ^k / k!을 계산하고, 여기에 (1 − (q/p)^(z−k))를 곱하여 합산한다. sum 변수의 초기값은 1.0으로 설정하여 k > z인 꼬리 항의 합을 미리 포함시킨 뒤, k ≤ z인 구간에서 차감하는 방식으로 무한급수를 유한한 계산으로 바꾼다. 이 수치적 변환이 백서에서 "무한급수 꼬리를 계속 더하지 않도록 식을 정리하고"라고 언급하는 부분의 구체적 구현이다.

수식과 코드를 함께 이해하면 비트코인 보안의 정량적 근거가 명확해진다. 공격자 해시 파워가 작을수록, 그리고 수취인이 기다리는 블록 수가 많을수록, 이중지불 성공 확률은 지수 함수적으로 감소한다. 이 수학적 확실성이 비트코인 프로토콜의 신뢰성을 뒷받침하는 핵심이다.
