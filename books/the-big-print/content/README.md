# content/

이 디렉토리에는 사람이 읽기 쉬운 정리 결과를 둔다.

예상 파일 예시:

- `en/book-summary.md`
- `en/chapters/01-introduction.md`
- `ko/book-summary.md`
- `ko/chapters/01-introduction.md`

권장 챕터 Markdown 필드:

- 제목
- 원문 출처 파일
- 핵심 요약
- 핵심 포인트
- 검수 상태

언어별 원칙:

- `en/` : 원문 기반 영어 정리본
- `ko/` : 영어 정리본을 기준으로 번역한 한국어 버전
- `ko/` 번역 시 `.codex/skills/bitcoin-translation-style/SKILL.md`의 용어/관점 규칙을 따른다.

가능하면 동일한 slug/file name을 유지해서 대응 관계를 쉽게 만든다.
