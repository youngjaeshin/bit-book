# 14 Full Node Concepts

## Chapter Digest

Full nodes are the basis for independent verification of bitcoin balances and transactions. It explains what a full node checks, what data it stores, why pruning changes storage without removing the need for trust in your own hardware, and how Core and Knots differ as client choices with slightly different policies and trade-offs.

The network side of the picture matters too. Initial block download, outbound and inbound connections, Electrum servers, and RPC interfaces all determine how the node fits into a personal setup and how much of the network the user can actually participate in. A node is not only a database; it is the user’s local proof that the ledger being used is real.

It closes by tying the technical pieces back to financial sovereignty. Running a node is not a hobby layer above Bitcoin; it is one of the ways a user stops asking permission from someone else’s server to know what their money is doing.
