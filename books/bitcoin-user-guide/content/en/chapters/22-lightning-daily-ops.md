# 22 Lightning Daily Ops

## Chapter Digest

Daily Lightning use starts with funding the node and opening channels with intention. The chapter walks through moving funds into the node’s on-chain wallet, searching for peers, adding them, and opening multiple channels so the user can see how capacity, routing relationships, and liquidity shape everyday usability.

The sequence matters because a Lightning node is not useful simply by existing. The guide shows that opening one channel is only the beginning, and that adding more channels, watching inbound liquidity, and choosing peers deliberately changes whether the node can actually receive and route payments in the way the operator needs.

By treating the node as a living wallet rather than a background service, the chapter turns Lightning into a practice of continual maintenance. The user learns to think in terms of channel balance, counterparty quality, and the kind of payment behavior the node is meant to support.
