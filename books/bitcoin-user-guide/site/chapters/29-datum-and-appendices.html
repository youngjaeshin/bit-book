29 Datum And Appendices

Chapter Digest

Datum is presented as a way to reconnect mining with the user’s own node and payout environment. On Umbrel, Bitcoin Knots has to be installed first, and Datum’s Bitcoin Address, Coinbase Tag, and collaborative reward-sharing settings then determine where value actually lands and how that mining identity is expressed. Mining is treated as a full operating stack, not just a hash-rate number.

The hardware examples make that stack concrete. Bitaxe Gamma 601 and Avalon Nano 3 are both wired into Datum by pointing Stratum Host at the local node, using port 23334, placing the payout destination in the User or Worker field, and leaving the password as x. But setup is only half the job. Status, Clients, Threads, and Coinbaser all have to be checked so the operator can verify that the gateway is alive, the miners are really attached, and the work is actually flowing.

Reward handling receives the same amount of attention. The guide contrasts simple receipt to a BlueWallet address with routing rewards toward an address generated by the user’s own Lightning node. It then extends into Lightning-aware pool rewards, where CLN Offers and signed messages are needed before payouts can be directed properly. The operator is expected to understand both how work is submitted and how money comes back.

The appendices widen the chapter from mining into recoverability. Mnemonic-generation algorithms on Keystone 3 Pro, SeedSigner, and BlueWallet are checked so seed words are treated as verifiable outputs rather than opaque magic. Recovery paths for four-letter fragments, bit-string backups, ordered word fragments, and the BIP-39 list then make the closing standard explicit: self-custody is incomplete unless the user can also reconstruct control under bad conditions.

요약

Datum은 채굴을 사용자의 자체 노드와 다시 묶는다. 엄브렐에서 먼저 비트코인 노츠를 설치해야 Datum을 쓸 수 있고, 이 가이드는 Bitcoin Address, Coinbase Tag, Collaborative reward sharing 설정을 조정해 지급 경로를 사용자의 통제 아래 두는 방식으로 설명한다.

하드웨어 예시도 구체적이다. 비트엑스 감마 601과 아발론 나노 3는 모두 Datum을 통해 연결되며, 로컬 노드 주소를 Stratum Host로 쓰고, 포트는 23334, 지급받을 주소를 User나 Worker로 넣고, 비밀번호는 x로 둔다. Status, Clients, Threads, Coinbaser 창을 확인하면서 게이트웨이가 제대로 붙었는지 점검한다.

보상 흐름도 채굴 흐름만큼 중요하다. 이 가이드는 블루월렛 주소나 라이트닝 노드의 온체인 주소로 보상을 받는 방법을 보여 주고, 오션 풀처럼 풀 채굴에서 라이트닝 보상을 원할 경우 CLN Offer 생성과 메시지 서명까지 연결한다.

부록은 키스톤 3 프로, 시드사이너, 블루월렛의 니모닉 생성 알고리즘을 비교하고, 4자리 영단어 일부만 남긴 백업, 비트 문자열 백업, 영단어 순서 백업, BIP-39 목록을 이용한 복구를 정리한다. 백업 단어는 검증 가능하고, 다시 구성 가능하고, 다른 장치로 옮길 수 있어야 한다는 것이 마지막 메시지다.

이 가이드는 버튼 이름과 메뉴 경로를 외우는 데서 끝나지 않는다. 어떤 설정을 왜 바꾸는지, 어떤 입력값이 어떤 동작을 여는지, 어디서 검증해야 실수가 줄어드는지까지 운영 관점으로 묶어 준다. 그래서 독자는 단순히 절차를 따라 하는 수준을 넘어, 다음에 비슷한 상황이 와도 스스로 점검하고 복구하고 선택할 수 있는 기준을 얻게 된다.
