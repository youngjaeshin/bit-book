# 4. Proof of Work — Basics

## Chapter Digest

Bitcoin's network contains miners who verify transactions and produce blocks. Miners collect only valid, double-spend-free transactions into a block and stack it on top of the existing chain. Because many miners compete to do this simultaneously, the network needs a system for reaching consensus on who added a block first. Bitcoin solves this with proof of work.

To understand proof of work, picture each block with a lock attached to its side. The combination that opens this lock is determined entirely by the transaction data inside the block. Change even one character of that data and the combination changes completely. This lock is a hash function. A miner tries a temporary value called a nonce, hashing it together with the block data, and keeps incrementing the nonce until the resulting hash value falls below a specific target. Finding that nonce demands enormous computation and electricity, yet verifying it requires only one hash function execution. Presenting the nonce proves the miner did the work — hence the term "proof of work."

This concept originates in Hashcash, a system designed in 1997 by Adam Back to combat spam email. Hashcash required a sender to find an input whose SHA-1 hash began with 20 zero bits — a search taking roughly one million hash computations on average. For an ordinary sender sending one email this cost is negligible, but for a spammer targeting ten thousand recipients it demands two hours and forty-five minutes of computation. Bitcoin's proof of work follows the same logic: miners vary the nonce in the block header until the SHA-256 hash of the header falls below the current target value.

When a miner finds a valid nonce, the block becomes a candidate for the next position in the chain. Receiving nodes verify the nonce in a single hash computation and, if valid, begin mining the subsequent block using this block's hash as their previous-block reference. The system is equivalent to one vote per CPU. IP addresses can be forged or acquired in bulk, but CPU computation carries real cost.

Altering data in an already-confirmed block requires redoing that block's proof of work from scratch, and then redoing the proof of work for every block that follows it. Once enough blocks accumulate, past data becomes practically immutable. An attacker attempting to alter a block ten positions back must redo proof of work for all ten blocks while honest miners continuously extend the honest chain — an insurmountable task.

Proof-of-work difficulty adjusts automatically every 2,016 blocks to keep average block production at one block per ten minutes. If those 2,016 blocks were mined faster than 20,160 minutes the difficulty rises; if slower it falls. This self-correcting mechanism means that no matter how much computing power advances in the future, Bitcoin's ten-minute block interval and its security remain intact.

As long as honest miners control the majority of hash power, the longest chain is always the honest chain. An attacker can only attempt double-spending — recovering coins they already sent — and even altering a single block costs hundreds of billions of won in electricity. Bitcoin achieves irreversible transactions without any trusted third party. Proof of work is the most essential mechanism in Bitcoin; not understanding it means not truly understanding Bitcoin itself.
