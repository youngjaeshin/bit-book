# Calculations

## Chapter Digest

The security argument is formalized by the calculations that follow. If an attacker tries to reverse history, their probability of success depends on their relative hash power and the number of confirmations they must overcome. That argument shows that Bitcoin’s security is not hand-waving confidence but something that can be reasoned about probabilistically.

This mathematical treatment matters because it turns decentralization from slogan into measurable robustness. The network is not secure because no attack is imaginable, but because certain attacks become overwhelmingly unlikely unless an attacker controls extraordinary resources. Calculation gives precision to the claim that Bitcoin is hard to cheat.
