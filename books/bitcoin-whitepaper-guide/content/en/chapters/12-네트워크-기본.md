# 5. Network — Basics

## Chapter Digest

Every device participating in the Bitcoin network is called a node. A node is any device that sends, receives, or relays information, including smartphones, desktops, and routers. As of 2024, approximately twenty thousand nodes are directly connected to the Bitcoin main network.

Bitcoin nodes fall into three main categories by function. A full node stores every block from the genesis block to the latest, and verifies transactions independently without relying on any external party. A simplified payment verification node (SPV node, or light node) is designed for devices with limited storage or bandwidth; it keeps block headers but borrows data from full nodes to verify transactions through a specialized method. A mining node collects valid transactions from the memory pool (mempool) into a block and performs proof of work. Mining pool operators must run full nodes, but individual miners participating in a pool do not need to.

When a transaction is broadcast, the network proceeds through six steps. First, the transaction is propagated to all nodes. When Alice signs a transaction sending bitcoin to Bob, that transaction data reaches the main network through her wallet app or her own full node. Each receiving node independently verifies the validity of inputs and outputs, the correctness of the signature, the absence of double-spending, and the compliance of data size. Valid transactions enter the node's mempool and are forwarded to neighboring nodes; invalid ones are discarded immediately. Second, mining nodes collect transactions from the mempool and assemble a block. Third, each mining node performs proof of work on its block, varying the nonce until the block's hash falls below the target. Fourth, the first mining node to complete proof of work propagates its block to neighboring nodes. Fifth, receiving nodes verify both the block's proof of work and the validity of every transaction inside it. Sixth, mining nodes that accept the block begin building the next block using this block's hash as their previous-block reference — their implicit vote of acceptance.

When two mining nodes far apart on the network complete proof of work almost simultaneously, a fork occurs. Each group of nodes mines atop the block it received first, splitting into two camps. This tie breaks as soon as either camp finds the next block. The convergence back to a single chain is called a re-org, and the discarded block is called a stale block. Bitcoin's ten-minute block interval exists specifically to keep stale block frequency low; shorter intervals increase forks and paradoxically slow final settlement.

Not every node must receive every transaction. If most mining nodes have received a transaction, it will soon be included in a block. A node that misses a block discovers the gap when the following block arrives, requests the missing block from neighbors, and fills it in. A block whose predecessor cannot be found is called an orphan block — a distinct concept from a stale block.

This entire process is Bitcoin. No node needs permission from a trusted third party; each independently verifies transactions and blocks and independently selects the longest chain. This consensus mechanism is called Nakamoto Consensus. It operates on four pillars: decentralization, majority-rule principle, difficulty adjustment, and incentives. Before Bitcoin, reaching such consensus without a trusted third party was considered impossible. Bitcoin's invention of Nakamoto Consensus is one of the most profound breakthroughs in the history of monetary systems.
