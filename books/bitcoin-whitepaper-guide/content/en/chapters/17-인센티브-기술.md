# 6. Incentive — Technical

The technical section of Chapter 6 examines the actual data structure of coinbase transactions and verifies at the code level that bitcoin genuinely had no pre-mining.

## The Actual Data Structure of a Coinbase Transaction

Every block contains multiple transactions, each assigned a sequential number. The zeroth transaction — always the first — is the coinbase transaction. Unlike ordinary transactions, the coinbase transaction references no prior transaction. Its input (vin) field can contain up to 100 bytes of arbitrary data chosen by the miner. Its output (vout) field contains the miner's wallet address and the block reward amount.

The genesis block's coinbase transaction is hard-coded directly into the Bitcoin Core source code. Its txid is `4a5e1e4baab89f3a32518a88c31bc87f618f76673e2cc77ab2127b7afdeda33b`. The coinbase input field contains a message encoded in hexadecimal. Decoded, it reads: "The Times 03/Jan/2009 Chancellor on brink of second bailout for banks" — a headline from the British newspaper The Times on the day the genesis block was mined. This serves simultaneously as a timestamp proving when the block was created and as a commentary on the failures of the existing financial system. The output is 50 BTC locked to Satoshi Nakamoto's public key in P2PK format.

Block 854,689 is the first block whose coinbase input contains the message "Made in USA." Miners and mining pool operators can embed any message they choose in the coinbase input field, and many do so to mark significant events.

## Don't Trust, Verify: Did Bitcoin Really Have No Pre-mining?

Bitcoin is open source — the full codebase is publicly available. The genesis block is not received from another node over the network; it is hard-coded into Bitcoin Core itself. The relevant code is found in `src/kernel/chainparams.cpp`, lines 41–78. Lines 63–65 contain this comment:

> Build the genesis block. Note that the output of its generation transaction cannot be spent since it did not originally exist in the database.

Full nodes validate all transactions in each block and store unspent outputs in the UTXO set so they can be quickly referenced when future transactions attempt to spend them. In `src/validation.cpp`, lines 2481–2487 contain this comment and corresponding code:

> Special case for the genesis block, skipping connection of its transactions (its coinbase is unspendable)

For the genesis block alone, the transactions are not added to the transaction database. If anyone attempts to spend the genesis coinbase reward, full nodes find nothing in the UTXO set and reject the transaction as invalid. The transaction can never propagate through the network.

Anyone running a full node can verify this directly using the `bitcoin-cli getrawtransaction` command. Querying the genesis block coinbase txid returns:

```json
{
  "result": null,
  "error": {
    "code": -5,
    "message": "The genesis block coinbase is not considered an ordinary transaction and cannot be retrieved"
  },
  "id": null
}
```

The genesis block coinbase is not in the database and cannot be retrieved. This confirms at the code level that bitcoin had absolutely no pre-mining. Satoshi Nakamoto did not set aside any coins for himself before releasing bitcoin to the world.

## Fee Units Before and After SegWit

Before SegWit, transaction fees were denominated in SATS/B (satoshis per byte). After SegWit, fees are denominated in SATS/vB (satoshis per virtual byte). If the fee rate is 3 SATS/vB and a transaction is 100 vB in virtual size, the fee is 300 SATS. Current recommended fee rates can be checked in real time at mempool.space. When transferring bitcoin from an exchange to a personal wallet, excessively high fees are typically the exchange's own service charge, not the Bitcoin protocol fee.

## Summary

The coinbase transaction is the sole source of new bitcoin supply, and its structure can be inspected directly in real block data. The genesis block's coinbase reward is permanently unspendable by explicit design in the Bitcoin Core codebase, proving at the code level that no pre-mining occurred. Bitcoin's guiding principle — "don't trust, verify" — is demonstrated most powerfully here: anyone can inspect the source code and the blockchain data to confirm for themselves that the system is fair.
