Chapter 9 Intuitive Understanding — Combining and Splitting Value

Bitcoin uses the UTXO (Unspent Transaction Output) model. Each UTXO is like a banknote or check with a specific denomination written on it. Unlike an account-balance model where you can withdraw any amount from your balance, a UTXO cannot be split. To pay 1,000 satoshis when you only hold a 10,000-satoshi UTXO, you must spend the entire UTXO as input and create two outputs: one of 1,000 satoshis to the recipient and one of 9,000 satoshis back to yourself as change — just as you hand over a 10,000-won note and receive 9,000 won in change.

Multiple UTXOs can be combined into a single new UTXO, or a single UTXO can be spent to produce multiple outputs that become several new UTXOs. A transaction's input field accepts multiple UTXOs, and its output field can contain multiple recipients. Wallet applications typically show only the total balance and automatically select appropriate UTXOs when constructing a transaction, though some wallets allow manual coin selection.

Holding UTXOs that are too large exposes your approximate balance every time you spend, which hurts privacy. Fragmenting them too finely means more inputs per transaction and therefore higher fees. A practical guideline is to split UTXOs into amounts you expect to spend over two weeks to one month — around 100,000 to 1,000,000 satoshis in 2024 conditions.

The difference between a transaction's total input value and its total output value is the miner fee. There is no dedicated fee field in a transaction; the miner simply claims the remainder. If you spend a 10,000-satoshi UTXO and write only a 1,000-satoshi output, the remaining 9,000 satoshis automatically go to the miner. You must explicitly include a change output addressed back to yourself to recover the difference.

A seemingly alarming consequence of UTXO chains is that verifying one transaction appears to require verifying every ancestor transaction all the way back to the coinbase — an exponentially expanding fan-out. In practice, this is not a problem. Because every transaction is already verified by the entire network when it enters the mempool, propagates, gets included in a block, and as the block propagates, you only need to confirm that the immediately preceding transaction genuinely exists in a block and that the new transaction's inputs are valid. The network has already done the rest. There is no need to extract a fully independent copy of the entire transaction history; every transaction is transparently public and held by tens of thousands of full nodes.

9장 직관적 이해 — 금액 합치기와 나누기

비트코인은 UTXO(미사용 거래 출력) 모델이다. UTXO 하나하나는 금액이 다르게 적힌 지폐나 수표와 같다. 잔고 모델에서는 잔고에서 원하는 만큼 꺼낼 수 있지만, UTXO는 쪼갤 수 없다. 1만 SATS짜리 UTXO를 천 원짜리 버스 요금에 쓰려면 만 원짜리 지폐를 내고 9천 원을 거슬러 받듯이, 입력값으로 1만 SATS UTXO를 사용하고 출력값으로 1천 SATS와 9천 SATS 두 개의 새 UTXO를 만들어야 한다.

여러 개의 UTXO를 합쳐 하나의 새로운 UTXO를 만들거나, 하나의 UTXO를 사용해 여러 출력값을 갖는 거래를 만들어 새 UTXO 여러 개로 쪼갤 수 있다. 거래의 입력에는 여러 개의 UTXO가 들어갈 수 있으며, 출력도 여러 개가 될 수 있다. 지갑 앱들은 보통 사용자에게는 UTXO 금액의 총합만 보여주고, 송금 시 자동으로 적합한 UTXO를 선택해 거래를 만들어 준다.

UTXO 금액이 너무 크면 거래할 때 자신의 보유량이 노출되어 프라이버시에 좋지 않고, 너무 잘게 쪼개면 한 번의 거래에 많은 입력이 필요해 수수료가 증가한다. 권장 방식은 하나의 UTXO당 2주에서 한 달 정도 사용할 양으로 분할하는 것이다.

비트코인 거래의 입력값 총합은 출력값 총합보다 항상 크거나 같다. 그 차이가 채굴자에게 지불하는 수수료다. 거래 데이터 자체에 수수료 항목이 따로 없으므로, 잔돈을 돌려받고 싶다면 자신에게 보내는 출력을 반드시 적어야 한다. 그렇게 하지 않으면 입력과 출력의 차액 전부가 채굴자의 수수료가 된다. 1만 SATS를 입력으로 사용하고 1천 SATS만 출력으로 적으면, 9천 SATS가 자동으로 채굴자에게 간다. 수수료는 금액이 아니라 데이터 크기에 비례하므로, 입력이나 출력이 많아질수록 수수료가 늘어난다.

거래 검증에서 팬아웃 구조가 문제가 되지 않는 이유도 중요하다. 새로운 거래를 검증하기 위해 그 이전의 모든 거래를 코인베이스 거래까지 소급하며 검증할 필요는 없다. 입력값에 사용된 바로 직전 거래가 블록에 실제로 존재하는지, 입력값 총액이 출력값 총액보다 큰지, 잠금 스크립트가 올바르게 해제되는지만 확인하면 된다. 그 이전 거래가 블록에 실렸다는 것은 이미 네트워크 전체가 그 거래를 검증했다는 뜻이기 때문이다. 거래 이력의 완전히 독립된 사본을 추출할 필요가 없으며, 모든 거래는 투명하게 공개되어 있고 수만 개의 풀 노드가 보관하고 있다.
