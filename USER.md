# USER.md

## 이 프로젝트에서 사용자가 원하는 것

- `epub/`와 이후 `pdf/`에 넣어둔 책 원본을 바탕으로 내용을 정리하고 싶다.
- 책별로 사람들이 읽을 수 있는 **HTML 기반 학습 페이지**를 만들고 싶다.
- 각 책에는 최소한 다음이 포함되길 원한다.
  - 책 소개
  - 챕터별 또는 섹션별 요약
  - 챕터별 영어 Markdown 정리본
  - 챕터별 한국어 Markdown 번역본
  - 핵심 포인트 정리
  - 간단한 퀴즈
- 결과물은 한 권만 처리하는 일회성 작업이 아니라, **여러 책에 반복 적용 가능한 구조**여야 한다.

## 현재 우선순위

1. 프로젝트 구조를 먼저 안정적으로 셋업한다.
2. `The Big Print.epub`를 테스트 책으로 사용한다.
3. EPUB 기준 파이프라인을 먼저 만들고, PDF는 후순위로 둔다.
4. 여러 에이전트가 들어와도 `plan.md`와 `progress.md`만 읽으면 바로 이어서 작업할 수 있게 한다.
5. 책 내용은 가능하면 **영문판과 한글판 두 버전**으로 관리한다.
6. 서문/헌사/추천사 같은 front matter는 기본적으로 제외하고, 저자가 나눈 실제 챕터 기준으로 정리한다.
7. PDF도 챕터 구분이 명확하면 같은 목표 구조로 처리한다.
8. 한국어 번역은 오스트리안 경제학과 비트코이너 용어/관점을 반영한다.

## 작업 스타일 선호

- 설명은 한국어로, 짧고 명확하게 한다.
- 먼저 구조와 워크플로우를 잡고 그 다음 구현한다.
- 저작권/배포 리스크를 항상 먼저 고려한다.
- 자동 생성 결과는 검수 가능한 중간 산출물로 남긴다.
